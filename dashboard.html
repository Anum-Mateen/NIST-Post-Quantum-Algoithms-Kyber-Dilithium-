03 — Algorithms

How Each Algorithm Works

Both are built on Module-LWE in polynomial rings Zq[X]/(XN+1), but they serve different cryptographic purposes.

CRYSTALS-Kyber
Key Encapsulation Mechanism · FIPS 203 · Kyber-512
1Key Generation — Sample secret s and error e. Compute public key t = As + e from random matrix A.
2Encapsulate — Encrypt a random message using the public key. Output: ciphertext (u, v) + shared secret.
3Decapsulate — Use private key s to recover the message from ciphertext: w = v − sᵀu → shared secret.
Use Case
TLS key exchange · VPNs · Secure channels
CRYSTALS-Dilithium
Digital Signature Scheme · FIPS 204 · Dilithium2
1Key Generation — Sample small secret vectors s₁, s₂. Public key t = As₁ + s₂.
2Sign — Sample masking vector y. Compute challenge c from high bits of Ay. Signature z = y + cs₁ with hint bits.
3Verify — Recompute Az, check high bits match challenge hash, verify norm bound on z.
Use Case
Document signing · Firmware auth · Certificates

05 — Analysis

Key Findings

Interpretation of your benchmark results — useful for the IEEE report analysis section.

🚀

Why Kyber is faster

Kyber performs one matrix multiply per operation. Signing builds a 1024-element challenge from polynomial coefficients plus up to 10 fixed-point iterations — structurally heavier.

⏱

Why Sign is slow

Dilithium's signing constructs a large challenge string, runs convergence iterations on w₁, and checks hint-bit norms — all using O(N²) schoolbook polynomial multiplication.

📐

O(N²) vs NTT

This educational implementation uses schoolbook poly-mul (65,536 ops per multiply). Production implementations use NTT — O(N log N) — achieving a 10–100× speedup on the same hardware.

🔒

Security model

Kyber achieves IND-CCA2 (adaptive chosen-ciphertext secure). Dilithium achieves EUF-CMA (existentially unforgeable). These are the gold standards for KEM and signature schemes.

📦

Key size trade-off

Dilithium's ~4× larger keys embed more information for signature binding. Both are practical — Kyber replaces RSA-2048 in TLS; Dilithium replaces ECDSA in PKI.

🌐

Complementary, not competing

Kyber handles confidentiality (key exchange), Dilithium handles authenticity (signatures). A complete post-quantum system uses both — they are designed to work together.

⚠ Implementation Note

This is an educational implementation using naive O(N²) poly-mul and toy hash functions. For production, use liboqs, PQClean, or the NIST reference implementations. Relative ratios between algorithms are valid; absolute timings are not representative of production performance.
